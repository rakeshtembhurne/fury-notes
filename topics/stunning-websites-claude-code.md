# 3 Ways to Build Stunning Websites with Claude Code

## Key Insights

- **Iterate using HTML files first** — HTML is much easier to modify than React/Next.js during rapid iteration. Claude Code can make targeted changes without a full framework build cycle. Use HTML for initial design exploration before committing to a framework.

- **SuperDesign (superdesign.dev) is the "Cursor for design"** — An interactive canvas for real-time UI iteration with AI. Previously required its own API key, but now integrates directly with your existing Claude Code subscription. Watch AI create/modify UI iterations in real time.

- **ShadCN provides beautiful, customizable, accessible UI components** — Works with React and Next.js. New preset feature lets you experiment with component styles before importing into your project. Better to have a design ready before going to Claude Code to avoid repeated revision cycles.

- **Google Stitch for design + Gemini 3 Pro** — Stitch now integrates Gemini 3 Pro Thinking and Nano Banana for AI-enhanced design. Use it to generate designs with AI, then pass to Claude Code for implementation.

- **Color palette first with Coolors** — Before designing, establish your color theme. Coolors lets you experiment with multiple palettes side-by-side and export in your preferred format. Color decisions made early prevent revision cycles later.

## Summary

Build stunning sites with Claude Code by: (1) iterating in HTML first, (2) using SuperDesign canvas for real-time visual iteration, (3) using ShadCN presets and Google Stitch for design preparation, and (4) establishing color palette early with Coolors. The key insight: design first, then code — iterate visually before going to Claude Code.
