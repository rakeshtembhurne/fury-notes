# Micro-Startup Playbook — Key Learnings

## Synthesis from Twitter Data

Data source: 5,964 tweets from 473 accounts (indie hackers, AI founders, micro-startup builders)

---

## The Playbook

### 1. Build Fast, Ship Faster

| Approach | Who | Result |
|----------|-----|--------|
| Vibe coding | @marclou | 33 startups |
| Ship in hours | @starter_story | $100K/mo potential |
| Codex from phone | Kid founder | v0.001 shipped |

**Rule:** Don't overthink. Ship v0.001, iterate.

### 2. Portfolio > Single Product

@marclou: 33 startups running
@levelsio: $10M ARR, multiple products
@marclou: Multiple acquisitions on TrustMRR

**Rule:** Many bets, diversified risk.

### 3. AI-First Products

- AI-ready docs (`.MD` for agents)
- AI agents that work while you sleep
- Browser-native AI agents
- Agent swarms for scale

**Rule:** If AI can't use it, you're behind.

### 4. Build to Sell

| Product | MRR | Sold | Multiple |
|---------|-----|------|----------|
| Reading Club | $700 | $12,000 | 17x |
| Slack Bot | $300 | $10,000 | 33x |
| Reddit Scraper | $72 | $2,000 | 28x |

**Rule:** Build with exit in mind. TrustMRR = easy sales.

### 5. Build in Public

- Share progress on Twitter
- Vibe coding thread = engagement
- "1 billion HTTP requests" = social proof
- Acquisition posts = credibility

**Rule:** Twitter = marketing. Use it.
