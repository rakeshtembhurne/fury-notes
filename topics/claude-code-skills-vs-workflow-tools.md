# Anthropic's Skills Strategy — Skills vs. Workflow Tools

## Key Insights

- **Skills 2.0 vs. Workflow Tools are complementary** — Skills teach Claude how YOU work (encoded preferences/workflows). MCPs and other workflow tools extend what Claude CAN do (capabilities). Both are needed for maximum effectiveness.

- **Skills are plain English instructional manuals** — A skill is not code or a prompt. It's a written specification that teaches Claude to follow your exact process, format, and standards every time — without you re-explaining it.

- **The 3 Skills 2.0 superpowers: EVALs, Benchmark, Triggers** — EVALs tests your skill against realistic scenarios. Benchmark checks if your skill is still needed after model upgrades. Triggers diagnose why your skill might not be firing.

- **Two types of skills: Capability Uplift vs. Encoded Reference** — Capability uplift patches genuine model weaknesses (generic outputs). Encoded reference captures your specific workflow when the model is already decent at the task. Build differently for each type.

- **Don't be generic in skill definitions** — "Write me great copy" is not a skill. Use Claude to extract the structure of your own expertise before building. Ask: "What are the key decisions and rules I should define for a skill that does X?" — use AI to figure out what to teach AI.

## Summary

Skills 2.0 makes Claude learn your specific way of working. MCPs extend capabilities. Skills + MCPs together create the most powerful Claude Code setup. Build skills by encoding your actual rules, then use EVALs/Benchmark/Triggers to verify they're working and still relevant after model updates.
