# 25 Hidden n8n Features That Save Hours of Work

## Key Insights

1. **Pin Data for API Cost Savings** - Use the pin data feature to cache API responses so you don't incur repeated costs during workflow development. Once you get successful output from an expensive API call (like Perplexity), pin it to reuse the data while building the rest of your workflow without re-running the API.

2. **Use Mock Data for Testing** - Fill any node with test data using the pencil icon to simulate inputs without running the full workflow. This allows you to test downstream nodes independently and iterate quickly on your workflow logic.

3. **Auto-Mapping Variables** - Instead of manually dragging connections between nodes, n8n can automatically map variables between connected nodes when you hover over a field, making workflow construction faster.

4. **Variable Collections** - Store reusable variables globally in your workflow. Instead of hardcoding values in multiple places, define them once and reference them throughout, reducing maintenance overhead and making updates easier.

5. **Execute Workflows from Webhook Testing** - Use the iframe embed option to allow external stakeholders to trigger and test your webhooks without giving them full n8n access, useful for client demos and collaboration.
